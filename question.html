Question 1: Discuss of var, let, and const

var

The "var" keyword declares a variable that is scoped to its current execution context, optionally initializing it with a value.

let

The "let" keyword declares a variable with block scope, optionally initializing it with a value.

const

The "const" keyword declares constants, which are block-scoped like variables defined using "let," but the value of a constant cannot be changed. The "const" declaration creates a read-only reference to a value.

Question 2: Tell us the use cases of null and undefined

If we don't assign a value to a variable when declaring it, the JavaScript compiler assigns it the 'undefined' value. Therefore, we can check if the variable contains a null or undefined value, which means we have an uninitialized variable.

Question 3: What do you mean by REST API?

"Representational State Transfer (REST)" is an architectural style that defines a set of constraints for creating web services. A REST API is a simple and flexible way to access web services without any additional processing.

REST technology is generally preferred over the more complex Simple Object Access Protocol (SOAP) technology because REST uses less bandwidth, is simpler and more flexible, making it better suited for internet usage. It is used to retrieve or provide information from a web service. All communication through REST API exclusively utilizes HTTP requests.

Operation: A request is transmitted from the client to the server in the form of a web URL, using either an HTTP GET, POST, PUT, or DELETE request. Subsequently, a response is returned from the server in the form of a resource, which can take various formats such as HTML, XML, images, or JSON. Presently, JSON is the most widely used format in web services.
